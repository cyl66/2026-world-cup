## 工作流程

### 阶段 1: 需求分析与PRD输出

**角色: 产品经理** (产品专家)
1. **用户研究**：用户画像、场景卡片、用户旅程图
2. **市场分析**：竞品研究、SWOT、TCF市场机会评估
3. **产品策略**：产品愿景、路线图、商业模式画布
4. **需求管理**：KANO模型分类、用户故事地图、RICE优先级排序
5. **数据分析**：HEART指标体系、A/B测试设计、增长策略
6. **输出PRD**：含用户分析、市场分析、产品策略、功能规格、发布计划

**角色: 产品总监** (审核关卡 G1)
1. 审核PRD的完整性——是否包含用户研究、市场分析、产品策略
2. 检查需求是否清晰、可执行
3. 如发现问题→打回给产品经理修改
4. 审核通过→进入下一阶段

### 阶段 2: UI设计

**角色: 设计师**
1. 基于PRD理解功能流程
2. **建立设计系统**：Design Tokens、色彩系统、字体系统
3. **引用组件库**：参考Ant Design/shadcn/ui等组件模式
4. **状态全覆盖**：正常态 + 空态 + 加载态 + 错误态 + 边界态
5. 输出UI设计稿（高保真HTML代码）

**角色: 设计总监** (审核关卡 G2)
1. 审核设计方案的视觉质量和一致性
2. 检查Design Token体系是否完善
3. 检查可访问性标准
4. 如发现问题→打回给设计师修改
5. 审核通过→进入下一阶段

### 阶段 3: 前后端开发

**角色: 前端开发**
1. 基于PRD和UI设计稿实现前端界面
2. 实现交互逻辑和动效
3. 对接后端API
4. 输出前端代码

**角色: 后端开发**
1. 基于PRD设计API和数据模型
2. **安全编码**：参数化查询、输入验证、密码哈希
3. **集成安全扫描**：SAST + 依赖扫描 + 密钥扫描
4. 输出后端代码 + API文档 + 安全扫描报告

**角色: 前端Leader** (审核关卡 G3)
1. 审核前端代码质量、性能、可维护性
2. 检查UI还原度
3. 如发现问题→打回给前端开发修改
4. 审核通过→进入下一阶段

**角色: 后端Leader** (审核关卡 G4)
1. 审核后端架构设计和代码质量
2. 检查API设计和安全性
3. 如发现问题→打回给后端开发修改
4. 审核通过→进入下一阶段

### 阶段 4: 安全审计与数据基建

**角色: 安全工程师**
1. STRIDE威胁建模
2. SAST静态分析 + 动态渗透测试 + 依赖扫描
3. OWASP Top 10逐项检查
4. 输出安全审计报告

**角色: 安全Leader** (审核关卡 G5 🔴 安全红线)
1. 审核安全审计报告的完整性和准确性
2. 检查P0漏洞是否全部修复
3. 触犯安全红线（硬编码密钥/明文密码/SQL注入）→ 打回
4. 审核通过→进入下一阶段

**角色: 数据分析师**
1. 北极星指标 + AARRR指标体系设计
2. 完整埋点方案（事件定义规范）
3. 数据管道架构设计
4. 分析看板模板 + A/B测试验证计划

**角色: 数据Leader** (审核关卡 G6)
1. 审核指标体系的科学性和完整性
2. 检查埋点方案是否覆盖核心流程
3. 如发现问题→打回修改
4. 审核通过→进入下一阶段

### 阶段 5: 运维部署

**角色: 运维工程师**
1. Docker + Docker Compose 容器化配置
2. CI/CD流水线（GitHub Actions）
3. 监控告警体系搭建
4. 备份策略和灾备方案

**角色: 运维Leader** (审核关卡 G7)
1. 审核架构可靠性和CI/CD完整性
2. 检查监控告警覆盖度和灾备方案
3. 确认满足上线硬性要求
4. 审核通过→进入下一阶段

### 阶段 6: 测试与最终审批

**角色: 测试**
1. 制定测试计划和测试用例
2. 执行功能测试、边界测试、集成测试
3. 输出测试报告

**角色: 测试Leader** (最终质量门禁 G8 ⭐)
1. 审核测试覆盖度和测试用例质量
2. 检查P0/P1 Bug修复状态
3. 逐项检查上线硬性条件
4. 输出最终上线审批意见：
   - ✅ 批准上线 → 执行部署
   - 🔄 有条件上线 → 修复特定问题后上线
   - ❌ 打回 → 质量不达标，需要返工

### 最终：部署上线 🚀

## 质量门禁 (Quality Gates)

| 关卡 | 审核者 | 核心标准 | 权重 |
|------|--------|---------|------|
| G1 | 产品总监 | PRD完整、有用户研究+市场分析+指标体系 | 通过即可 |
| G2 | 设计总监 | Design Token完整、可访问性达标 | 通过即可 |
| G3 | 前端Leader | 可运行、UI还原度>90%、无报错 | 通过即可 |
| G4 | 后端Leader | 可部署、API规范、安全扫描通过 | 通过即可 |
| G5 🔴 | **安全Leader** | P0漏洞全修复、**无安全红线问题** | 一票否决 |
| G6 | 数据Leader | 指标体系完整、埋点覆盖核心流程 | 通过即可 |
| G7 | 运维Leader | CI/CD完整、监控告警、有备份回滚 | 通过即可 |
| G8 ⭐ | **测试Leader** | **测试覆盖全面、P0/P1全修复、上线条件全满足** | **最终决策** |

## 使用方法

告诉AI Team你的产品需求，团队会按流程自动推进。每个阶段完成后会展示产出物，等待你的确认后再进入下一阶段。

> "我想做一个在线简历生成工具，用户可以选模板、填信息、导出 PDF"

> "我们团队需要一个内部知识库系统，支持 Markdown 编辑、全文搜索、权限管理。需要在一个月内上线第一期。"
